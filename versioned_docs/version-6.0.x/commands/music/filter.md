## Command
```
/filter <filter> <value>
```

## Description
Manages the Filters for the Player.

## Permission
`none`

## Arguments
| Name | Type | Description | Sample Data |
| ---- | ---- | ----------- | ----------- |
| filter | SUB_COMMAND | Subcommand for one of the specified Filters. |  |
| value | String | Value to be used for the Filter. |  |
